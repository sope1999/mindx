# mindx-exe 需求文档 v1.0

---

## 一、软件定位

mindx 是一款 **.md 文件关系可视化工具**，主要面向使用 OpenClaw 进行开发的用户。启动基文件夹后，通过文件树、目录树图、引用树图、依赖图四个维度展示项目结构及文件间引用关系。

目标：**开源项目** → GitHub 发布 → 打包为 .exe，开箱即用。

---

## 二、分发与安装

### 2.1 分发形式
- 打包为单个 **.exe** 文件（PyInstaller）
- 在 GitHub 上开源，通过 Release 页面下载

### 2.2 安装
- 像普通软件一样，用户选择安装目录
- 程序文件 + 数据文件（config.yaml 等）全部存放在安装目录下
- 不在项目文件夹内遗留任何 mindx 专属文件
- 不在系统目录（%AppData%、%ProgramFiles%）写入

### 2.3 升级
- 启动时静默检查 GitHub Release 新版本
- **不弹窗**，仅在更新按钮旁显示红点或数字标识有新版
- 点击更新按钮后下载新 exe 覆盖旧 exe
- **升级不清除用户数据**（已有项目列表、配置、localStorage 位置数据）

---

## 三、首次启动体验

- 启动后界面与当前 mindx 一致（三面板布局 + 顶部标签栏）
- 无项目时：标签栏仅显示 **「+ 添加项目」** 按钮，作为引导入口
- 有项目时：加载已保存的项目列表，**自动选中列表第一个**项目

---

## 四、多项目管理

### 4.1 项目存储
- 持久化保存，存储在安装目录 `config.yaml`
- 项目名 = 所选文件夹名，**不可在 mindx 内编辑**（如需改名，用户自行重命名文件夹，然后删除旧项目重新添加）

### 4.2 项目添加
- 点击「+ 添加项目」→ 弹出系统文件夹选择器 → 选择目标文件夹
- 文件夹名自动作为项目名
- 若添加的文件夹名与已有项目重名，自动追加后缀 `project`、`project(2)`、`project(3)` ...

### 4.3 项目删除
- 标签栏项目卡片右上角 × 删除按钮
- **需要确认弹窗**：「确定要删除项目"[项目名]"吗？删除后可从项目列表重新添加。」
- 确认后从列表移除

### 4.4 项目切换
- 顶部标签栏方式切换
- 最多显示 4 个标签，超出部分收入 **下拉列表**
- 下拉列表显示所有项目（包括标签栏中的 4 个）
- 切换项目时：
  - 停止旧项目的文件监听
  - 启动新项目的文件监听
  - 立即全量刷新一次数据（避免监听间隙导致过期）

### 4.5 项目路径失效
- 如果已保存的项目文件夹被删除或移动：
  - 标签页显示红色/警告图标
  - 点击该标签时提示："路径不存在：[path]，请重新选择文件夹或删除项目"
  - 提供「重新选择」和「删除项目」两个按钮

## 五、全局配置

### 5.1 位置
```
<安装目录>\config.yaml
```

### 5.2 配置结构（v1）
```yaml
projects:
  - name: "coder"
    root: "C:/SOFT/AI/coder"
    # links: ...        # 预留，暂不启用
    # file_types: ...   # 预留，暂不启用

version: "1.0.0"
port: 5020
```

### 5.3 作用范围
- 仅存储项目列表 + 全局首选项，不侵入项目目录

---

## 六、端口处理

- 默认端口：**5020**
- 端口被占用时：**弹窗让用户手动指定端口号**，重新启动服务
- 不自动切换端口

---

## 七、文件监听

### 7.1 监听范围
- 仅监听**当前选中项目**
- 监听文件类型：**`*.md`**（预留扩展能力，当前不做可选配置）
- 监听内容：文件新增、修改、删除、重命名

### 7.2 切换行为
- 切换到其他项目时：
  - 停止旧 watcher
  - 启动新 watcher
  - 立即全量扫描新项目目录（防止切换窗口期遗漏变更）

### 7.3 空文件夹
- 目录树图中**显示空文件夹节点**（即使不包含 .md 文件）
- 保持目录结构完整

---

## 八、三视图架构

### 8.1 面板总览

```
┌───────────┬─────────────────────────┬──────────────┐
│           │  [记忆树图]  [依赖图]    │              │
│  文件树    │                         │  文件摘要面板 │
│           │                         │              │
└───────────┴─────────────────────────┴──────────────┘
```

| 面板 | 模式 | 功能 |
|------|------|------|
| **文件树**（左） | 目录模式 | 纯文件夹层级排列 .md 文件 |
| | 引用树模式 | 按引用关系组织层级（当前不变） |
| **记忆树图**（中-Tab1） | 目录模式 | 纯文件夹层级树形布局图 |
| | 引用树模式 | 引用关系图（当前不变） |
| **依赖图**（中-Tab2） | 依赖图 | 全量引用边（`[→](path)` 解析） |
| **文件摘要**（右） | — | 选中文件时显示详情 |

### 8.2 文件树

#### 目录模式（改动项）
- 按文件夹层级排列，纯目录结构
- 每个 .md 文件作为一个叶子节点，不混入引用信息
- 保留文件分类标识（核心/基础/独立/外部 颜色标签）
- 保留陈旧检测标记（黄色"可能过期"）

#### 引用树模式（保留不变）
- 当前引用关系树，不改动

### 8.3 记忆树图

#### 目录模式（改动项）
- **树形布局**（vis.js hierarchical，direction: 'UD'）
- 从项目根目录开始，逐层向下展开子文件夹和 .md 文件
- **显示根节点**（项目名/文件夹名）
- 同一层级的文件夹/文件从左到右排列
- **无弹回问题**：树形布局天然层级清晰，节点不可拖动（或允许拖但会复位）
- 保留分类颜色标记

#### 引用树模式（保留不变）
- 当前引用关系图
- 当前物理布局 + 首次冻结后任意拖动

### 8.4 依赖图（保留不变）
- 全量引用边展示
- 保留双向箭头（A↔B 互相引用时）
- 保留分类颜色 + 节点自由拖动 + 位置保存
- 解析两类链接：
  - **内部链接**：项目内 .md 文件间的 `[→](path)` 引用
  - **外部链接**：引用到项目外部的文件，标注为外部

### 8.5 文件摘要面板（保留不变）
- 文件路径（完整路径 + 相对路径）
- 修改时间
- 被引用次数（父引用 + 子引用列表）
- 分类设置（可手动修改并持久化）
- 陈旧警告（列出具体过期的引用）
- 事件日志（实时文件变更通知）
- 同步建议

---

## 九、联动交互

- **三视图联动选中**：点任意视图的文件节点，其他视图高亮对应节点
- **节点拖动**：引用树模式和依赖图中节点可任意拖动，位置持久化到 localStorage
- **位置持久化**：引用树 + 依赖图的位置按项目分别存储
- **过滤联动**：文件树的分类过滤器与记忆树图、依赖图联动（预留）

---

## 十、链接规则

### 10.1 当前规则（v1）
- 统一使用 `[→](路径)` Markdown 链接格式
- 由 `parser.py` 解析提取

### 10.2 未来扩展（预留）
- 每个项目可配置自己的链接规则
- 通过"项目管理页面"设置（UI 上的独立页面）
- 支持的链接格式示例：
  - `[[wikilink]]`
  - `[text](relative/path)`
  - 代码级引用 `.include("file")`

---

## 十一、现有高级功能（全部保留）

| 功能 | 说明 |
|------|------|
| **文件分类** | 核心/基础/独立/外部 四类，颜色标签可视化 |
| **陈旧检测** | mtime 对比，引用者比被引用者旧则标记"可能过期" |
| **被引用计数** | 显示某文件被多少文件引用 |
| **分组节点** | 引用树中自动分组（工具链/项目/归档） |
| **分类设置** | 右键面板可手动修改分类并持久化 |
| **双向箭头** | A↔B 互引时显示双箭头 |
| **隐式引用** | `implicit-refs.json` 加载逻辑保留 |
| **同步规则** | 基于 MEMORY.md 收尾检查的规则保留 |
| **通知点击** | 陈旧/变更/同步通知可点击弹出详情 |

---

## 十二、文件解析

### 12.1 范围
- 默认解析 `*.md` 文件
- 预留 `file_types` 配置项扩展（.txt, .json, .yaml, .yml 等）

### 12.2 解析内容
- 引用链接：`[→](path)` 格式
- 隐式引用：`memory/implicit-refs.json`（如存在）
- 文件修改时间：用于陈旧检测
- 文件内容：用于摘要预览
